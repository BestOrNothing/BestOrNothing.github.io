Dependency parsing

My notes of dependency parsing from NLU course.

For a projective dependency tree, it can be described as a context-free grammar.
Projective dependency tree means that there is no cross between any two edges.

Graph-based dependency parsing as tagging

Goal: find the highest scoring dependency tree in the space of all possible trees for a sentence.

Since each word has exactly one parent, this is like a tagging problem. But when we solve this tagging problem, we need to think about the constraint of tree that there is no cycle.

The score of a dependency edge (i,j) is a function s(i,j). Then the score of a dependency tree y for sentence x is:

$$s(\mathbf x, \mathbf y) = \sum_{(i,j)\in \mathbf y}s(i,j)$$

Our task is to find the highest score for a given sentence x.
This task can be achieved using the following approach:

  • start with a totally connected graph
  • find the maximum spanning tree
  • use Chu-Liu-Edmonds to find it in $O(n^2)$, but it is not guaranteed to be projective

Chu-Liu-Edmonds (CLE) algorithms:

  • each node $j$ in the graph greedily selects the incoming edge with the highest score $s(i,j)$
  • If a tree result, it is the maximum spanning tree. If not, there must be a cycle.
  • Identify the cycle and contract it into a single node and recalculate scores of incoming and outgoing edges.
  • Then we greedily select edges again.

The score function is some sort of softmax.
